Usage

SATDBailiff can be used to to generate new datasets from Git repositories. When supplied a list of repositories and a target database configuration, SATDBailiff will crawl each commit in the target repository and output all SATD instances impacted by those commits to the specified database.

Setup

SATDBailiff requires a few dependencies to run:

If running from pre-compiled binaries:
  • Java 1.8+
  • A MySQL instance (local or remote)
If building from source:
  • Java 1.8+
  • A MySQL instance (or custom output implementation)
  • Maven 3

Running From Jar

SATDBailiff can be run as a JAR file using the following command:

java -jar SATDBailiff-1.0.jar -r repos.csv -d db.properties

Execution requires at least the two inputs:

  • -r: A CSV list containing repository URIs. The first entry should be the repository's URI, and the second can be an optional terminal commit hash. (Example)
  • -d: A .properties file containing db configuration. (Example)

Other inputs are detailed in the tool's help menu:

                  usage: satd-analyzer
 -a,--diff-algorithm  <ALGORITHM>   the algorithm to use for diffing (Must
                                   be supported by JGit):
                                   - MYERS (default)
                                   - HISTOGRAM
 -d,--db-props  <FILE>              .properties file containing database
                                   properties
 -e,--show-errors                  shows errors in output
 -h,--help                         display help menu
 -i,--ignore  <WORDS>               a text file containing words to ignore.
                                   Comments containing any word in the
                                   text file will be ignored
 -l,--n_levenshtein <0.0-1.0>      the normalized levenshtein distance
                                   threshold which determines what
                                   similarity must be met to qualify SATD
                                   instances as changed
 -p,--password  <PASSWORD>          password for Github authentication
 -r,--repos  <FILE>                 new-line separated file containing git
                                   repositories
 -u,--username  <USERNAME>          username for Github authentication

Building From Source

SATDBailiff is a Maven project, so familiarity with Maven is recommended to rebuild the tool.

In order to use the implicitly available SATD classification feature, you must manually include the Jar file for SATD Detector under 'lib/' in the root directory of the project, specified in SATDBailiff's pom file.
