Usage

SATDBailiff can be used to to generate new datasets from Git repositories. When supplied a list of repositories and a target database configuration, SATDBailiff will crawl each commit in the target repository and output all SATD instances impacted by those commits to the specified database.

Setup

SATDBailiff requires a few dependencies to run:

If running from pre-compiled binaries:
  • Java 1.8+
  • A MySQL instance (local or remote)
If building from source:
  • Java 1.8+
  • A MySQL instance (or custom output implementation)
  • Maven 3
  • You will also need to manually include the binaries for SATD Detector under 'lib/' in the root directory of the file, specified in SATDBailiff's pom file.

Running

SATDBailiff can be run as a JAR file using the following command:

java -jar SATDBailiff-1.0.jar -r repos.csv -d db.properties

Execution requires at least the two inputs:

  • -r: A CSV list containing repository URIs
  • -d: A .properties file containing db configuration. (Example)

Other inputs are detailed in the tool's help menu:
